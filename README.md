## 개요
- C# 프로그래밍 학습
- 1줄 소개: 사용자의 키보드 입력을 실시간으로 처리하고 관리하는 에코 메신저 프로그램입니다.

	- string 클래스의 Trim() 메서드를 이용한 데이터 정제와 IsNullOrEmpty를 활용한 유효성 검사s
	- DateTime 클래스로 실시간 전송 시각을 구하는 기능을 구현하였습니다.

## 실행 화면 (과제1)
- 과제1 코드의 실행 스크린샷

![과제1 실행화면](img/step1_demo.gif)

- 과제 내용
  - Label(표시), TextBox(입력), Button(전송), ListBox(대화창)를 적절히 배치합니다. 
  - 전송 버튼 클릭 시 TextBox의 텍스트를 ListBox의 항목(Items)으로 추가합니다. 추가 직후 TextBox의 내용을 비워(Clear) 다음 입력을 준비합니다.

- 구현 내용과 기능 설명
  - 사용자 입력을 위한 txtInputMessage 텍스트 박스와 전송을 위한 btnSendMessage 버튼, 그리고 대화 내역을 표시하는 lstEchoWindow 리스트 박스를 유기적으로 연결하여 에코 메시지 전송 로직을 완성하였습니다. 
  - 버튼 클릭 시 입력된 문자열을 리스트 박스 아이템에 즉시 추가하며, Clear() 메서드로 입력창을 초기화함과 동시에 Focus() 메서드를 호출하여 사용자가 마우스 조작 없이도 즉시 다음 메시지를 연속해서 입력할 수 있도록 UX 측면의 편의성을 극대화하였습니다. 
  - 메시지 양이 리스트 박스의 높이를 초과할 경우 스크롤바가 자동 생성되도록 구성하여 이전 내역 확인이 용이하도록 설계하였습니다.

- 사용한 기술과 구현한 기능:
  - string 클래스를 이용한 사용자 입력 데이터 처리
  - Items.Add() 메서드를 활용한 데이터 동적 추가

## 실행 화면 (과제2)
- 과제2 코드의 실행 스크린샷

![과제2 실행화면](img/step2_demo.gif)

- 과제 내용
  - 메시지 입력 후 엔터키(Enter)를 누르면 전송 버튼을 클릭한 것과 동일하게 동작하도록 구현합니다.
  - 메시지 전송 후 별도의 마우스 조작 없이도 입력창에 자동으로 포커스가 유지되어 연속적인 입력이 가능하도록 합니다.

- 구현 내용과 기능 설명
  - txtInputMessage 컨트롤의 KeyDown 이벤트 핸들러를 활용하여 사용자가 키보드의 엔터키를 눌렀을 때를 실시간으로 감지하도록 설계하였습니다. 
  - `e.KeyCode == Keys.Enter` 조건을 만족할 경우, 기존에 구현된 btnSendMessage의 로직을 중복 작성하지 않고 `btnSendMessage.PerformClick()` 메서드를 호출함으로써 코드의 재사용성을 높이고 전송 버튼과 동일한 동작을 수행하도록 완성하였습니다. 
  - 전송 직후에는 Focus() 메서드를 통해 커서를 입력창에 고정시켜 사용자가 흐름이 끊기지 않고 다음 메시지를 즉시 작성할 수 있도록 UX(사용자 경험)를 개선하였으며, 엔터 입력 시 발생하는 시스템 비프음을 방지하기 위해 `e.SuppressKeyPress = true` 설정을 추가하는 디테일을 반영하였습니다.

- 사용한 기술과 구현한 기능:
  - KeyDown 이벤트를 이용한 특정 키 입력 감지 및 처리
  - PerformClick() 메서드를 활용한 이벤트 강제 발생 기술

## 실행 화면 (과제3)
- 과제3 코드의 실행 스크린샷

![과제3 실행화면](img/step3_demo.gif)

- 과제 내용
  - 메시지 전송 시 현재 시간을 대화 내용 앞에 결합하여 출력하도록 구현합니다.
  - 입력된 문자열의 앞뒤 공백을 자동으로 제거하는 정제 과정을 추가합니다.
  - 별도의 라벨(lblStatus)을 통해 현재까지 리스트박스에 누적된 메시지의 총 개수를 실시간으로 표시합니다.

- 구현 내용과 기능 설명
  - `DateTime.Now.ToString("HH:mm:ss")` 형식을 활용하여 메시지가 전송되는 시점의 정확한 시, 분, 초 데이터를 추출하고, 이를 사용자 메시지와 결합하여 출력함으로써 대화의 시간적 맥락을 파악할 수 있도록 구현하였습니다. 
  - 사용자가 실수로 입력할 수 있는 불필요한 앞뒤 공백을 제거하기 위해 `String.Trim()` 메서드를 적용하여 데이터 저장소인 ListBox에 정제된 데이터만 입력되도록 처리하였습니다. 
  - 또한, 메시지가 추가될 때마다 `lstEchoWindow.Items.Count` 속성값을 참조하여 하단 lblStatus 라벨의 Text를 즉시 업데이트함으로써, 현재까지 수신된 전체 메시지의 양을 사용자가 실시간으로 확인 가능하도록 상태 동기화 로직을 완성하였습니다.

- 사용한 기술과 구현한 기능:
  - DateTime 클래스를 이용한 현재 시간 정보 추출 및 문자열 포맷팅
  - String.Trim() 메서드를 활용한 입력 데이터 정제 기술
  - Items.Count 속성을 활용한 UI 데이터 실시간 반영

## 실행 화면 (과제4)
- 과제4 코드의 실행 스크린샷

![과제4 실행화면](img/step4_demo.gif)

- 과제 내용
  - 리스트박스의 데이터를 관리하기 위한 전체 초기화 및 선택 항목 삭제 기능을 구현합니다.
  - 사용자 편의성을 위해 입력창의 글자 수를 제한하고 실시간으로 상태를 표시하는 기능을 추가합니다.
  - 힌트 텍스트(Placeholder) 기능을 통해 사용자에게 직관적인 입력 가이드를 제공합니다.

- 구현 내용과 기능 설명
  - btnReset 버튼 클릭 시 Items.Clear() 메서드를 호출하여 리스트박스의 모든 내역을 초기화하고 상태 라벨을 0으로 동기화하는 로직을 완성하였습니다. 
  - TextChanged 이벤트를 활용하여 사용자가 입력하는 글자 수를 실시간으로 계산해 lblCharCount에 표시하며, 50자 초과 시 시각적 경고(색상 변경)와 함께 입력을 제한하여 데이터의 규격을 관리하도록 설계하였습니다. 
  - 또한 Enter와 Leave 이벤트를 제어하여 입력창 클릭 시 가이드 문구가 사라지는 플레이스홀더 기능을 구현하였으며, SelectedIndex 속성과 RemoveAt() 메서드를 사용하여 사용자가 선택한 특정 메시지만 골라 삭제할 수 있는 심화 관리 기능을 성공적으로 구현하였습니다.

- 사용한 기술과 구현한 기능:
  - SelectedIndex 및 RemoveAt을 활용한 특정 데이터 제어 기술
  - TextChanged 이벤트를 이용한 실시간 데이터 모니터링 및 UI 피드백
  - Enter/Leave 이벤트를 활용한 동적 텍스트박스 가이드 기능
